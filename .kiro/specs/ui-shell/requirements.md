# 共通レイアウト（UIシェル） — Requirements

> spec: `共通レイアウト（UIシェル） (ui-shell)`
> 配置想定: `.kiro/specs/ui-shell/requirements.md`
> 前提 steering: `product.md`・`tech.md`・`structure.md`
> 参照 UI: `Shisui Dashboard (standalone).html`（バンドル済みモック。ビジュアル参照のみ・原則は `ui-spec-reconciliation.md` に同じ）
> 関連: `.kiro/specs/ui-spec-reconciliation.md`（各画面 ↔ spec 対応表。本 spec はその対応表に無い「共通ヘッダー・サイドバー」を担う）

## 概要 (Introduction)

現行フロントエンドはページ間の共通レイアウトを持たず、トップ（`/`）は見出しのみでナビゲーション手段が無い
（各画面は URL 直接入力でのみ到達可能）。モックアップをビジュアル参照として、**全画面共通のヘッダー・
サイドバーナビゲーション**を新設し、配色・フォントのデザイントークンを導入する。
**データ取得・業務ロジックは変更しない**（既存の各ページコンポーネント・hooks・API は無変更）。

### スコープ (In Scope)
- 共通ヘッダー（ロゴ・アプリ名・サブタイトル・エッジPC稼働の静的プレースホルダー・「オンプレ LAN」バッジ）
- 共通サイドバー（ダッシュボード／AI学習／タスク＋「設定」展開でエッジPC／閾値／色マスター）
- ルーティング変更（`/` → `/dashboard` リダイレクト、`AppLayout` のネスト）
- デザイントークン（配色・フォント）の導入・JetBrains Mono の自前ホスト

### スコープ外 (Out of Scope)
- 各画面（ダッシュボード等）自体の見た目の作り直し（対応する各 spec で別途実施）
- エッジPC稼働数・タスク件数・モデルバージョンの実データ表示（実データ接続は各対応 spec 側）
- ユーザーアバター等、個人識別を前提とした要素（認証は単一共有 Basic 認証のためスコープ外）
- Noto Sans JP の自前ホスト（OS 標準フォントで代用）・カスタムスクロールバー装飾

## 要件 (Requirements)

### UI-R1. 共通ナビゲーション
**User Story**: 検査運用担当者として、どの画面からでも他の画面へ迷わず遷移したい。

**受け入れ基準 (EARS)**
1. 全画面で共通のサイドバーナビゲーションを表示する（SHALL）。
2. サイドバーは「ダッシュボード／AI学習／タスク」の3項目と、「設定」を展開すると表示される
   「色マスター／閾値／エッジPC」の3項目を持つ（SHALL）。
3. 現在表示中の画面に対応するナビ項目をアクティブ表示する（SHALL）。
4. 現在のルートが設定配下（色マスター／閾値／エッジPC）のいずれかの場合、設定サブメニューは
   初期状態で展開されている（SHALL）。

### UI-R2. トップページのリダイレクト
**受け入れ基準 (EARS)**
1. `/` にアクセスした場合、`/dashboard` にリダイレクトする（SHALL）。
2. 見出しのみの `Home` 画面は廃止する（SHALL）。

### UI-R3. 共通ヘッダー
**受け入れ基準 (EARS)**
1. 全画面でヘッダーにロゴ・アプリ名・サブタイトルを表示する（SHALL）。
2. エッジPC稼働状況は実データ接続を前提とせず、静的なプレースホルダー表示とする（SHALL）。
   稼働中を誤認させる配色・アニメーション（緑パルス等）は用いない（SHALL NOT）。
3. ユーザー個別のアバター等、単一共有認証と矛盾する要素は表示しない（SHALL NOT）。

### UI-R4. デザイントークン
**受け入れ基準 (EARS)**
1. 共通の配色・フォントを CSS 変数として定義し、共通レイアウトに適用する（SHALL）。
2. 日本語フォントは OS 標準フォントへのフォールバックチェーンとする（Noto Sans JP は自前ホストしない）
   （SHALL）。
3. 数値・英数字表示用に JetBrains Mono（400/500/600・Latin のみ）を自前ホストする（SHALL）。

### UI-R5. 既存画面との非干渉
**受け入れ基準 (EARS)**
1. 既存の各ページコンポーネント（`Dashboard` 等）は変更しない。共通レイアウトは `<Outlet/>` を通じて
   子画面をラップするのみとする（SHALL）。

## 確定事項・残課題 (Resolved & Pending)

確定（brainstorming にて利用者承認済み）:
- ナビ構成はモックアップ通り（3項目＋設定展開で3項目）。
- ヘッダーのエッジPC稼働・タスク件数バッジ・モデルバージョン表示は今回すべて非表示（実データ未接続）。
- ユーザーアバターは削除（認証モデルと不整合）。
- 日本語フォントは OS 標準で代用、JetBrains Mono のみ自前ホスト。
- カスタムスクロールバー装飾はスコープ外。

残: なし（実装着手可）。
