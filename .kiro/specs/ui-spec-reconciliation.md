# UI 画面案 ↔ spec 整合メモ

> 対象 UI: `Shisui_Dashboard.html`（バンドル済みモック）／ 対象 spec: `.kiro/specs/*`・steering
> 原則: **spec＝挙動・データ・粒度の正**、**UI＝ビジュアル/レイアウトの参照**。相違は **UI を spec に合わせる**。

| UI 画面 | 対応 spec | 状態 |
|---|---|---|
| ダッシュボード | 検査結果ダッシュボード（`dashboard-*`） | 対応 |
| 閾値設定/管理 | 閾値管理（`requirements/design/tasks`） | 対応 |

| 共通ヘッダー・サイドバーナビ | 共通レイアウト（UIシェル） (`ui-shell`) | 対応（2026-07-13 追加。`Shisui Dashboard (standalone).html` を参照して新規spec化） |

## 整合している点

- 粒度: UI は「日別」「JST」を使用 → **日次×フルタプル**統一と一致。
- ドメイン語: チェーン種別（金属・コイル・ビスロン）、色名、欠陥種別（糸ほつれ・異物・欠け・変色・キズ）、
  閾値スコープ（全色）など、spec を肉付けする具体が含まれる。
- エッジPC: UI の **1〜4号機（IP 付き）** が `エッジPC管理` の登録対象（名称・host）と一致。
- エッジPC監視: UI の稼働状態は **FTP 送信可否の監視**として spec 化（`エッジPC管理` E-R5）。
  **CPU/メモリ/ディスク/FPS は飾り**（テレメトリ取得は実装しない）。配信ポートは `model_port`（収集用ポートは対象外）。

## 相違点の反映状況（修正版 UI で確認）

1. **人名（鈴木・田中 等）**: 修正版で**削除済み** ✓（担当者/記名は持たない。コメント匿名・単一共有認証＝spec のまま）。
2. **「ロット番号」列**: 修正版で**削除・日次化済み** ✓（ロットは不使用。日次×フルタプルに統一）。
3. **「総合精度」指標**: 修正版で**削除済み** ✓（spec の4指標に揃えた）。
4. **取り込み状況画面**: **スコープ外（取り下げ）**。取り込み有無の確認は本アプリの役割でなく、台帳データも DB に無いため実装しない。

## 反映済み（号機フィルタ）

- **号機（検査機台）フィルタをダッシュボードの正式機能として追加**（修正版 UI の号機選択に対応）。
  - `検査結果ダッシュボード` の requirements（R1.3）・design（集計前 WHERE・API `machine_ids`・`/machines`）・tasks に反映済み。
  - **号機は集計前フィルタ**（既定 全号機）。メトリクス粒度は `日次 × フルタプル` のまま。閾値は色レベル（号機別閾値なし）。
  - **逸脱判定・昇格は全号機の日次集計**を用いる（号機で絞らない）＝ダッシュボード表示専用。
  - 号機 ＝ 検査機台 ＝ エッジPC（`エッジPC管理` の登録対象と一致）。
  - 棚卸し: 業者検査データの**機台識別列**の有無を確認（`inventory.sql` 手順2・手順9 に追加済み）。
